### #1 — Lógica de paginação ambígua em `fetch_cursor_pages()`

- **Arquivo:** `scripts/meta_collector.py:120-149`
- **Severidade:** CRITICAL
- **Descrição:** `fetch_cursor_pages()` chama `page.load_next_page()` na condição do `if` (linha 129) mas não reatribui o retorno a `page`. O SDK Meta retorna o próximo cursor via `load_next_page()`, mas aqui o resultado é descartado. O loop itera `page` novamente, causando re-processamento dos mesmos itens ou loop infinito dependendo do comportamento interno do SDK.

**Código atual:**
```python
def fetch_cursor_pages(cursor, max_retries: int = 5) -> list[dict]:
    """Iterate a cursor that supports direct iteration and has_next_page."""
    results = []
    attempt = 0
    page = cursor
    while True:
        try:
            for item in page:
                results.append(item.export_all_data())
            if not page.load_next_page():  # ← retorno descartado, page não muda
                break
        except StopIteration:
            break
        except FacebookRequestError as e:
            error_code = e.api_error_code()
            if error_code == 80000:
                wait = 2 ** attempt
                attempt += 1
                if attempt > max_retries:
                    raise
                print(f"  Rate limit. Waiting {wait}s...")
                time.sleep(wait)
            elif error_code in (190, 102, 2500):
                print(f"\nERROR: Invalid/expired token (code {error_code}). Update META_ACCESS_TOKEN in .env")
                sys.exit(1)
            else:
                raise
        except Exception:
            break
    return results
```

**Fix sugerido:**
```python
def fetch_cursor_pages(cursor, max_retries: int = 5) -> list[dict]:
    """Iterate a cursor that supports direct iteration and has_next_page."""
    results = []
    attempt = 0
    page = cursor
    while True:
        try:
            for item in page:
                results.append(item.export_all_data())
            # O SDK Meta modifica `page` in-place via load_next_page().
            # Se não há mais páginas, retorna False.
            if not page.load_next_page():
                break
            # Reset attempt counter após sucesso
            attempt = 0
        except StopIteration:
            break
        except FacebookRequestError as e:
            error_code = e.api_error_code()
            if error_code == 80000:
                wait = 2 ** attempt
                attempt += 1
                if attempt > max_retries:
                    raise
                print(f"  Rate limit. Waiting {wait}s...")
                time.sleep(wait)
            elif error_code in (190, 102, 2500):
                print(f"\nERROR: Invalid/expired token (code {error_code}). Update META_ACCESS_TOKEN in .env")
                sys.exit(1)
            else:
                raise
        except Exception:
            break
    return results
```

> **Nota:** Validar com dados reais qual é o comportamento do SDK — se `load_next_page()` modifica `page` in-place ou retorna novo cursor. A documentação do `facebook-business` SDK é ambígua nesse ponto.

### #11 — Retry não reseta cursor após backoff

### #12 — Sem `safe_float()` no parsing de insights

- **Arquivo:** `scripts/data_processor.py:84-88`
- **Severidade:** MEDIUM
- **Descrição:** `float(row.get("spend", 0) or 0)` funciona para `None` e strings numéricas, mas crasha com `ValueError` se a Meta API retornar string não-numérica (ex: `"N/A"`, `"--"`, `""`). O `or 0` não protege contra strings truthy não-numéricas.

**Código atual:**
```python
spend = float(row.get("spend", 0) or 0)
impressions = int(row.get("impressions", 0) or 0)
clicks = int(row.get("clicks", 0) or 0)
reach = int(row.get("reach", 0) or 0)
frequency = float(row.get("frequency", 0) or 0)
```

**Fix sugerido:**
```python
def safe_float(value, default=0.0):
    try:
        return float(value) if value else default
    except (ValueError, TypeError):
        return default

def safe_int(value, default=0):
    try:
        return int(float(value)) if value else default
    except (ValueError, TypeError):
        return default

spend = safe_float(row.get("spend"))
impressions = safe_int(row.get("impressions"))
clicks = safe_int(row.get("clicks"))
reach = safe_int(row.get("reach"))
frequency = safe_float(row.get("frequency"))
```

### #14 — Delta com NaN quando `prev_row` contém valores NaN

- **Arquivo:** `scripts/data_processor.py:196-204`
- **Severidade:** MEDIUM
- **Descrição:** `compute_delta()` retorna `None` quando `previous == 0`, mas não trata o caso onde `previous` é `NaN` (vindo de dias anteriores sem dados completos). `NaN` passa pela condição `== 0` sem match e `(current - NaN) / NaN` propaga NaN para as colunas delta.

**Código atual:**
```python
def compute_delta(current: float, previous: float) -> float | None:
    if previous == 0:
        return None
    return round((current - previous) / previous * 100, 2)
```

**Fix sugerido:**
```python
import math

def compute_delta(current: float, previous: float) -> float | None:
    if previous == 0 or (isinstance(previous, float) and math.isnan(previous)):
        return None
    if isinstance(current, float) and math.isnan(current):
        return None
    return round((current - previous) / previous * 100, 2)
```

- **Arquivo:** `scripts/data_processor.py:291-296`
- **Severidade:** LOW
- **Descrição:** Quando o CSV já existe, rows com datas iguais são silenciosamente removidas e substituídas pelas novas. Isso é o comportamento correto (idempotência), mas não há log informando quantas rows foram substituídas. Se dados do período anterior forem perdidos por erro, o usuário não tem como saber.

**Código atual:**
```python
if path.exists():
    existing = pd.read_csv(path, dtype=str)
    if "date" in existing.columns and "date" in df.columns:
        current_dates = df["date"].unique().tolist()
        existing = existing[~existing["date"].isin(current_dates)]
        df = pd.concat([existing, df.astype(str)], ignore_index=True)
```

**Fix sugerido:**
```python
if path.exists():
    existing = pd.read_csv(path, dtype=str)
    if "date" in existing.columns and "date" in df.columns:
        current_dates = df["date"].unique().tolist()
        replaced_count = existing[existing["date"].isin(current_dates)].shape[0]
        if replaced_count > 0:
            print(f"  INFO: Replacing {replaced_count} existing rows for dates: {current_dates}")
        existing = existing[~existing["date"].isin(current_dates)]
        df = pd.concat([existing, df.astype(str)], ignore_index=True)
```
